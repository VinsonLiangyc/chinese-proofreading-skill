# 中文校对 Skill · 适用于 Claude

你是否经历过这样的场景——每次写完稿子，自己费尽心思检查了 n 遍，发出去后突然有读者/用户给你留言：你这里有个错别字。

这完全不是你的粗心，这是我们大脑的工作方式——我们会自动脑补自己写的文字，越读越顺，越看越像是一点错别字都没有。慢慢地，我们甚至会觉得错的也是对的，毕竟这大段文字，我们已经对着反复修改了好多遍了。

专业出版社解决这个问题的方式是配一个校对编辑。但是，我们也许是个内容创作个体户，也许是某家企业的一位文案，没有一位专业的校对编辑来与我们一起工作。

同样作为内容创作者的我，也没有。

所以我做了这个 Skill，来帮助我自己以及大家。

---

## 它具体干什么

你把写好的中文文本丢给它，说一句「帮我校对」。它跑六项检查，返回四样东西：

**修订后全文** — 改完的版本，结构不动，文风不碰

**修改清单** — 改了哪里，一条一条列出来，没有暗改

**修改原因** — 每条改动为什么改，依据哪条规范

**体例一致性检查** — 全文扫一遍，看你是不是前半篇写「AI」后半篇又写「人工智能」，前面用「」后面用""

---

## 六项检查，对着国标来

| 查什么 | 依据 |
| --- | --- |
| 标点符号 | GB/T 15834—2011 |
| 数字写法 | GB/T 15835—2011 |
| 高频别字 | 《咬文嚼字》年度百大别字（松弛≠松驰，震撼≠震憾，宣泄≠渲泄） |
| 混淆词 | 20+ 组：营利/盈利、定金/订金、作/做、即/既…… |
| 中英夹用 | CY/T 154—2017 |
| 体例一致性 | 品牌名、术语、数字格式、引号风格、人称——全文统一 |

它只纠错和标注不一致。不帮你改写，不帮你润色，不会把你的文章变成另一篇文章。

---

## 谁会需要这个

**一个人做内容的创作者。** 公众号、小红书、播客文字稿、B站专栏——你写完就发，没人帮你过一遍。这个 Skill 就是你发布前的最后一道关。

**品牌和公关团队。** 新闻稿、产品发布文案、企业声明。官方渠道出一个错别字，它会被截图，会被转发，会被记住。多人协作的文件，体例一致性检查尤其有用——五个人写的稿子，合在一起格式乱了，它帮你兜底。

---

## 怎么触发

对 Claude 说以下任何一句：

> 「帮我校对」「校对一下」「有没有错别字」「发布前检查一下」「对外文案校对」「新闻稿校对」

不说就不启动。它不会在你不需要的时候跳出来纠正你。

## 安装

1. 打开 [claude.ai](https://claude.ai)，进入你的 Project
2. **Project Settings** → **Skills** → **Upload Skill**
3. 上传 `zh/SKILL.md`（中文界面）或 `en/SKILL.md`（英文界面）
4. 说「帮我校对」，搞定

```
chinese-proofreading-skill/
├── README.md            ← English
├── README.zh-CN.md      ← 这个文件
├── zh/
│   └── SKILL.md         ← 中文界面用
└── en/
    └── SKILL.md         ← English interface
```
